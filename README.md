# FC Backoffice

뱅크샐러드금융서비스 재무설계사를 위한 **KPT 기록 및 고객관리 차팅 서비스**입니다.

---

## 주요 기능

### 홈 (대화 로그)
- 오늘 상담한 고객 대화 로그 (날짜별 그룹, 최신 수정순)
- 고객 슬라이더로 자주 보는 고객 빠른 이동

### 고객 목록
- 전체 고객 리스트 (가나다순 / 최신연락순 / 단계순 정렬)
- 이름·방코드 검색
- 고객 추가 폼 (이름, 방코드, 전화번호, 생년월일, 대화 시작일, 이모지)
- 각 고객별 현재 진행 단계 칩 표시

### 고객 상세 — 단계별 상담 차팅

상담 진행 단계(니즈환기 → 문제인식 → 솔루션 → 청약 → 후속관리)별로 섹션이 나뉘며, 각 단계 데이터는 독립적으로 저장됩니다.

**니즈환기**
- 니즈 유형 칩 선택 (DB에서 동적 로드)
- 구체적 니즈 및 메모 자유 입력

**문제인식**
- 문제 유형 다중 선택 (카테고리별 그룹)
- 기존 보험 목록 및 문제 원인 태깅 (갱신형·보장범위·보장기간 등)
- 총 예산, 제안 방향, 메모 기록

**솔루션**
- 고객 예산 입력
- 담보 목록 추가 (자동완성 지원)
- 보험사별 심사결과 기록 (인수 / 부담보 / 거절)
- 고객 회신 상태 기록 (긍정 / 부정 / 무응답) 및 사유
- 최종 결정안 및 조정 결과 콤보 기록
- PDF 설계서 업로드
- 다음 상담 메모

**청약**
- 고객 확인사항 체크리스트 (주소·직업·병력 고지)
- 계약 고지사항 체크리스트 (해약환급금·보장·갱신형)
- 최종 상품명 및 보험료 기록

**후속관리 (CRM)**
- 후속 조치 체크리스트 (특약 삭제 등)
- 해피콜 안내사항 체크리스트 (청약철회·소멸시효 등)

**공통**
- 오늘 티키타카(대화 횟수) 기록
- 설계 동의 여부 체크
- KPT 메모 (Keep / Problem / Try)
- 알림 설정 (날짜·단계 지정)
- 발화 템플릿 연동 (단계별 템플릿 사용 및 고객 반응 기록)

### 인수 현황
- 전체 고객 인수/부담보/거절 이력을 한 테이블에서 조회
- 14개 컬럼 (병력명, 보험사, 상품, 심사결과, 부담보·할증 내용, 담보별 가입 여부, 설계년월 등)
- 행 클릭으로 수정, 고객별 필터 가능

### 캘린더
- 월별 상담 고객 현황 (날짜 셀에 고객명·단계 칩 표시)
- 알림 일정 표시

### 대시보드
- 오늘 목표 및 스크럼 작성
- 오늘 현황 통계 (신규·기존·총 상담수, 설계 동의수)
- 이번 주 일별 상담 현황 바 차트
- 퍼널 전환율 분석 (주간 / 직접 기간 설정 / 기간 비교)
- 전체 고객 퍼널 현황
- 일별 성과 리뷰 (KPT 회고 입력 및 뷰)
- 템플릿 반응률 TOP 5

### 발화모음 (템플릿)
- 단계별 발화 템플릿 등록·수정·삭제
- 공개/비공개 설정 (팀 내 공유)
- 태그 및 사용 맥락 기록
- 템플릿 수정 이력 관리
- 고객 상담 연동 시 사용 횟수 및 반응률 자동 집계

## 기술 스택

| 영역 | 기술 |
|------|------|
| Frontend | React 19, React Router v7 |
| Backend / DB | Supabase (PostgreSQL + Auth + RLS) |
| 배포 | Vercel |
| 스타일 | CSS Variables, Pretendard |
| 데스크탑 | Electron (선택) |

---

## 로컬 실행

```bash
npm install
npm start
```

`.env` 파일에 Supabase 환경변수 필요:

```
REACT_APP_SUPABASE_URL=...
REACT_APP_SUPABASE_ANON_KEY=...
```

## 배포

```bash
vercel --prod
```
